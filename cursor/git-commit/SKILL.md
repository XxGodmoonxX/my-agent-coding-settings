# 変更されたファイルをコミットする

## ステージ状態の確認

- まず `git status` を実行して、既にステージされている変更があるか確認してください。
- **ステージされている変更がある場合は、それらを優先的にコミットしてください。**
  - ステージされている変更の内容を確認し、適切なコミットメッセージでコミットしてください。
  - ステージされている変更のコミットが完了してから、未ステージの変更の処理に進んでください。

## 変更内容の確認

- 変更されたファイルの内容を確認してください

## コミットする単位の検討

- コミットするファイルの単位を検討してください。

## コミット

- コミットしてください。
  - まだトラックされていないファイルや設定ファイルなど、すべてのファイルがコミット対象です。
- コミットメッセージは Conventional Commits https://www.conventionalcommits.org/ja/v1.0.0 の仕様を参考にしたメッセージにしてください。
- 分割してコミットすべきと判断した場合は分割してコミットしてください。
